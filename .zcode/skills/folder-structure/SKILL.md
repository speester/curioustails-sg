# Project Folder Structure

## Configuration (Local Only — .gitignore)

```
config/
├── project-config.md              ← ALL variables & values (fill once per project)
├── business-context.md            ← Written by discovery skill (business facts + owner background)
└── expertise-library.md           ← Written by discovery skill (real stories, testimonials, E-E-A-T)
```

## Ignored Files

```
dist/                             ← Astro build output (not committed)
node_modules/                     ← NPM packages (not committed)
.astro/                           ← Build cache (not committed)
.env, .env.*.local                ← Environment variables (not committed)
```

---

## Summary

**In GitHub** (public):
- `src/`, `public/`, `astro.config.mjs`, `package.json`, `.gitignore`

**Local Only** (`.gitignore`):
- `config/` — Business & project config
- `.claude/` — Skills, documentation, settings
- `research/`, `briefs/`, `seo-audits/` — Planning & analysis

This structure ensures:
- Code is version-controlled (GitHub)
- Secrets & local work stay private
- Skills can reference stable paths without publishing them
