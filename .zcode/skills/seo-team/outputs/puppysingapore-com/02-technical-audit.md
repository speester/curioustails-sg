# Technical Audit: Puppy Singapore (2026-07-12)

Domain: puppysingapore.com | Stack: WordPress 6.9.4 + Rank Math + NitroPack (cache/CDN) + Cloudflare
Pages sampled directly this session: homepage, /all-available-puppies/, /dog-adoption-singapore/, /dog-adoption-process/, /dog-shelters/, /dog-adoption-fees/, /dog-adoption-challanges/, /dog-adoption-laws/, /project-adore/, /dog-adoption-event/, /fostering-dogs/, /preparing-for-adopted-dog/, /reviews/, /contact-us/, /blog/, /golden-retriever/, /corgi/, /british-bulldog/, /cavapoo/, /french-bulldog/, /shih-tzu/, /siberian-husky/, /pomeranian/, /dachshund/, /labrador/, /american-cocker-spaniel/, /american-cocker-spaniel-2/, /cavalier-king-charles/ (28 pages), plus robots.txt and all 3 sub-sitemaps.

## Health score: 58/100

The crawl foundation is genuinely solid — no noindex on any money page, clean robots.txt, real 404s, shallow one-click internal linking, and exceptional performance (Lighthouse 99/100 even on the heaviest page). But the money-page layer (30+ breed pages, the actual sales surface) is undermined by a template-level duplicate meta description bug, a broken sitemap file, near-total absence of image alt text, no Product/FAQ schema, and a dual-H1 defect replicated across an entire content cluster. These are fixable in a handful of template edits, not dozens of one-off fixes — which is why the score isn't lower.

## Minor issues

- Bare `/sitemap.xml` returns a 404 instead of redirecting to `/sitemap_index.xml`. Low impact since robots.txt correctly declares the real sitemap location, but a 1-line redirect would tidy this up.
- `/dog-shelters/` is served by plain WordPress (generator tag reads "WordPress 6.9.4," not "NitroPack" like every other page sampled) and shows 4 render-blocking scripts / 3 render-blocking stylesheets — it appears to be excluded from the NitroPack cache layer that's giving every other page its 99/100 performance score. Worth checking the NitroPack cache-exclusion rules.
- URL slug typo: `/dog-adoption-challanges/` (missing the "e" in "challenges"). Cosmetic; not worth a URL change now, just don't propagate the typo into new content or internal anchor text.
- Two stray nav/text links point to `https://puppysingapore.com/cavalier-king-charles` without the trailing slash (vs. the correct `/cavalier-king-charles/` used elsewhere). WordPress will auto-301 these, so functional risk is low — just an avoidable extra redirect hop on 2 links.
- Lighthouse's new "Agentic Browsing" category scored 33/100 on `/all-available-puppies/`. Not an urgent fix, but worth keeping on the radar as AI-agent traffic and answer-engine crawling grow.

## What's already good

- Lighthouse Performance is 99/100 on both the homepage and the heaviest page on the site (`/all-available-puppies/`, 1.1MB DOM, 1,500+ nodes) — FCP ~346ms, LCP ~856ms, CLS ~0.003. NitroPack + Cloudflare are doing real work here; don't touch this.
- No indexability blockers anywhere in the 28-page sample: every page returns `index, follow`, a correct self-referencing canonical (aside from the one intentional dedup pair), and real 404s (no soft-404s).
- Internal linking architecture is shallow and efficient: all 30+ breed pages sit one click from the homepage via primary navigation, and the adoption content cluster is densely cross-linked via "Related Posts."
- Mobile viewport meta is present and correctly configured on every page checked.
- The adoption content cluster isn't actually thin — 1,700–1,800 words with genuine, distinct FAQ content per page. The `low_content_rate` flag is a markup-bloat artifact, not a writing problem.
- robots.txt is clean, well-commented, and correctly scoped (blocks WP admin/system paths, allows the assets Google needs).

## Fix order

1. Fix the `page-sitemap.xml` 504 — this blocks sitemap-based discovery of nearly every money page and has no dependency on anything else below.
2. Fix the Rank Math description template pulling "Adaptability" as the meta description — one template fix likely resolves 30+ duplicate breed-page descriptions at once.
3. Fix the dual-H1 defect in the adoption-post template — one template fix resolves it across all ~10 adoption pages.
4. Restructure `/all-available-puppies/` headings: move individual puppy names off H2, consolidate the repeated "Contact Us for..." H3 boilerplate into one component.
5. 301 redirect `/american-cocker-spaniel/` → `/american-cocker-spaniel-2/`.
6. Bulk-add image alt text, starting with the 30+ breed pages and `/all-available-puppies/`.
7. Add FAQPage schema to the adoption pages (content already exists), fix the Person/Organization entity type site-wide, fix the relative-URL JSON-LD bug on Contact Us; discuss Product/Offer schema for breed pages with the owner given pricing is gated behind WhatsApp.
8. Shorten the `/dog-adoption-singapore/` (118 chars) and `/dog-shelters/` (90 chars) titles.
9. Housekeeping: redirect bare `/sitemap.xml`, fix the 2 trailing-slash nav links, check NitroPack cache-exclusion on `/dog-shelters/`.
